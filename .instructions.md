# redact/ -- Agent Context Warning

## WARNING: This directory has TWO independent git histories

```
C:\PROJECTS\LogicWizards\               <-- monorepo (LogicWizards/core)
└── SOLUTIONS/DevOps/SIDE-PROJECTS/
    └── tools/modules/redact/
        └── .git/                       <-- INNER repo (wwwizards/redactor)
```

The monorepo tracks files here as **plain blobs** (not a registered submodule).
This means:

- `git commit` in the monorepo root captures file content -- but does NOT
  advance `wwwizards/redactor`'s HEAD.
- `git status` at the monorepo root will show changes as modified files, NOT
  as a submodule diff. This looks normal and gives no warning.
- `wwwizards/redactor` can silently lag behind for an entire session.

**Before declaring any work here DONE, you must commit + push BOTH repos:**

```powershell
# 1. Commit to wwwizards/redactor (canonical upstream)
cd C:\PROJECTS\LogicWizards\SOLUTIONS\DevOps\SIDE-PROJECTS\tools\modules\redact
git add <files>
git commit -m "vX.Y.Z - <description>"
git push origin main

# 2. Commit monorepo pointer (so monorepo reflects latest content)
cd C:\PROJECTS\LogicWizards
git add SOLUTIONS/DevOps/SIDE-PROJECTS/tools/modules/redact/
git commit -m "bump redact to vX.Y.Z"
```

**Quick check before starting any session here:**

```powershell
cd C:\PROJECTS\LogicWizards\SOLUTIONS\DevOps\SIDE-PROJECTS\tools\modules\redact
git log --oneline -3        # what does the upstream remote actually have?
git remote -v               # confirm you are in the INNER repo context
```

If HEAD shows `Initial commit` or anything older than what you just edited,
the upstream is stale. Commit and push before doing anything else.
